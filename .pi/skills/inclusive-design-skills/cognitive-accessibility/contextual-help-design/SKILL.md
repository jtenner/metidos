# Contextual Help Design

Use this skill when designing assistance inside tasks and workflows. Focus on making help available exactly when needed, without cluttering the interface or forcing users to switch context.

## Assess Help Needs

- Identify steps where users commonly hesitate, make errors, or abandon tasks
- Look for jargon, hidden rules, or expectations that require prior knowledge
- Consider when examples, definitions, checklists, or confirmations would reduce confusion
- Avoid overwhelming users with all help at once

## Design Better Support

- Place help near the decision or action it supports
- Use progressive disclosure for details, examples, and secondary explanations
- Write help in plain, direct language with concrete examples
- Make recovery guidance actionable when something goes wrong

## Output

- Recommend where contextual help should appear
- Suggest formats such as inline text, expandable tips, examples, or guided steps
- Prioritize help that improves task completion and confidence
