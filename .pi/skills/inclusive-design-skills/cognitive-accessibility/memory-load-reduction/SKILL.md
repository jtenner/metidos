# Memory Load Reduction

Use this skill when reviewing journeys that depend on short-term memory or remembering previous instructions. Focus on making important information visible when needed, rather than expecting users to recall it.

## Identify Recall Burdens

- Look for instructions shown once and then hidden later
- Flag tasks that require remembering codes, rules, pricing, or prior choices across steps
- Check whether users must compare information across screens without support
- Consider memory difficulties related to age, fatigue, stress, or cognitive disability

## Reduce Memory Demands

- Keep key information visible at the point of action
- Provide summaries, checklists, and persistent references across steps
- Use autofill, saved preferences, and smart defaults where appropriate
- Support review and confirmation before final submission

## Output

- Highlight places where memory burden may cause errors or abandonment
- Recommend changes that support recognition and reference
- Prioritize improvements in multi-step and high-stakes flows
