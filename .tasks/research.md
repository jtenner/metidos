# Research

- Add a new doc in the docs folder
- Summarize the content at the top
- Add detailed structured content below the summary
- Use diagrams if necessary
- Use code blocks for code samples
- If the doc is about a migration of some kind, describe the current state of things and the desired state of things
- If the doc is about a new feature, describe the problem it solves and how it solves it
- If the doc is about a new design, describe the problem it solves and how it solves it
- If the doc is about a new API, describe the problem it solves and how it solves it
- If the doc is about a new architecture, describe the problem it solves and how it solves it
