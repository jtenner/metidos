# Ollama Model Provider Plugin

This copyable example registers an Ollama model provider family for Plugin System v1. It demonstrates multi-instance provider configuration loaded from plugin-owned `.data`, local-only network allowlists, model discovery, and stable provider/model identifiers.

## Files

- `metidos-plugin.json`: manifest and permission declarations.
- `index.ts`: provider registration, `.data` configuration parsing, Ollama discovery, and execution.
- `seed/providers.json`: first-activation seed copied by Metidos into `.data/providers.json`.
- `README.md`: author-facing setup and validation notes.
- `.data/`: generated by Metidos at runtime; do not commit.
- `.logs/`: generated only if plugin logging is enabled; do not commit.

Root `node_modules/` is forbidden. Keep imports local or `@metidos/plugin-api` only.

## Validation

From the Metidos repository, run:

```bash
bun test src/bun/plugin/examples.test.ts src/bun/plugin/manifest.test.ts
```

For a local smoke test after approval, keep Ollama running on an allowed local URL and confirm the provider appears in the model selector as `Ollama Model Provider / Ollama / <configuration>`.

## `.data` contents

`.data/providers.json` is the only expected durable file. It contains:

- `providers[]`: provider configurations.
- `id`: stable configuration id; changing it changes the Metidos provider id.
- `baseUrl`: local Ollama URL. The manifest currently allows only `http://localhost:11434/**` and `http://127.0.0.1:11434/**`.
- `discoverModels`: when true, startup refresh attempts `GET /api/tags` and returns an empty model list on failure.
- `models[]`: explicit static model declarations used only when `discoverModels` is `false`.

Do not store API tokens in this file. Local Ollama normally does not need a secret.

## Safe `.data` inspection

Prefer read-only inspection:

```bash
jq . .data/providers.json
```

Do not edit `.data/providers.json` while the plugin sidecar is running. Disable the plugin or stop Metidos first.

## Safe `.data` repair

1. Disable the plugin or stop Metidos.
2. Back up `.data/providers.json` if it contains local edits.
3. Repair malformed JSON or restore from `seed/providers.json`.
4. Restart/re-enable the plugin and review diagnostics if provider discovery still fails.

## Reset behavior

Reset Plugin Data moves `.data` to `.data-bak-{timestamp}`, recreates `.data`, copies `seed/providers.json`, and reloads the plugin. Local provider edits are lost unless copied back intentionally.

## Secrets and logs

This example should not handle secrets. Avoid adding Authorization headers or remote Ollama URLs unless the manifest `network.allow` and this guidance are updated in the same review.

## Embeddings and vector search

This copyable provider example should be updated carefully if it is adapted to expose embeddings. A provider that returns embedding vectors must declare `metidos:provides_embeddings`, implement `embed(context, request)`, and mark embedding-capable models with `api: "embeddings"` or `compat.providesEmbeddings: true`. It should not request `metidos:can_embed` or `metidos:lancedb` unless it also consumes embeddings or stores vectors itself.
