# Diverse User Needs

Use this skill when broadening design thinking beyond a single “average user.” Focus on how different abilities, contexts, and constraints shape people’s experience of the same interface.

## Expand the Lens

- Consider motor, visual, auditory, cognitive, language, and temporary impairments
- Include situational constraints such as glare, noise, one-handed use, low bandwidth, stress, or multitasking
- Look for assumptions about prior knowledge, confidence, speed, or device quality
- Identify which user groups are likely excluded by the current design

## Design More Inclusively

- Prefer solutions that work well across multiple needs rather than edge-case add-ons
- Reduce avoidable dependency on a single sense, input type, or cognitive strategy
- Provide flexibility where needs vary meaningfully
- Treat accessibility as product quality, not optional accommodation

## Output

- Describe which user needs are and are not supported
- Suggest changes that widen usability across contexts
- Prioritize inclusive improvements with broad benefit and high user impact
