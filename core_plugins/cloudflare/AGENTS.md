# AGENTS for Cloudflare Workers AI

## Purpose

First-party provider plugin for Cloudflare Workers AI. It registers Plugin System v1 model providers for chat completions and embeddings against Cloudflare's account-scoped OpenAI-compatible API at `https://api.cloudflare.com/client/v4/accounts/{account_id}/ai/v1`.

## Source layout

- `metidos-plugin.json`: v1 manifest reviewed by the local operator; declares `CLOUDFLARE_ACCOUNT_ID`, `CLOUDFLARE_API_TOKEN`, matching plugin settings, account-scoped network access, and the `cloudflare` / `cloudflare_embeddings` model providers.
- `index.ts`: plugin entry point that discovers Workers AI text-generation and text-embedding models from the Cloudflare account model search API, returns OpenAI-compatible chat provider configurations with per-configuration `piAuth` records, and provides embeddings through the OpenAI-compatible `/embeddings` endpoint.
- `AGENTS.md`: this guide.
- `.data/`: generated plugin data owned by Metidos; this plugin does not intentionally create files there.
- `.logs/`: generated plugin logs if enabled; discovery warnings may be logged there.
- `.data-bak-*`: generated reset backups; do not commit.

Root `node_modules/` is forbidden.

## Validation

1. Validate `metidos-plugin.json` against the Metidos Plugin Manifest v1 schema.
2. Run `bun test src/bun/plugin/cloudflare-core-plugin.test.ts` from the repository root.
3. Confirm no root `node_modules/` exists.
4. Confirm imports are local or `@metidos/plugin-api`.

## `.data` contents

This plugin has no durable plugin-owned data. If `.data/` exists, it is runtime-created by Metidos and can be regenerated by Reset Plugin Data.

## Safe `.data` inspection

- Prefer read-only inspection.
- Do not copy or print secret-bearing files if future changes add local credentials.
- Do not edit `.data` while the plugin sidecar is running unless this guide is updated with an explicit repair workflow.

## Safe `.data` repair

There are no hand-repairable data files today. Prefer Metidos Reset Plugin Data for unknown `.data` contents; it should be safe because this plugin has no seed data or durable local state.

## Reset behavior

Metidos Reset Plugin Data moves `.data` to `.data-bak-{timestamp}`, recreates `.data`, and restarts/reloads the plugin. No local configuration is recreated because the account ID and API token live in plugin settings or declared environment variables.

## Secrets and logs

Secrets are provided through the optional secret plugin setting `api_token` or the secret env var `CLOUDFLARE_API_TOKEN`. The Cloudflare account ID is not treated as a secret, but avoid exposing it unnecessarily in public logs. Do not log API tokens, Authorization headers, prompts, embedding inputs/vectors, model responses, or request payloads. Discovery failures may log bounded status/error messages only.

Credential and account precedence for discovery, inference, and embeddings is plugin settings first (`account_id`, `api_token`), then environment (`CLOUDFLARE_ACCOUNT_ID`, `CLOUDFLARE_API_TOKEN`).

## Embeddings and vector search

This plugin provides embedding vectors through the `cloudflare_embeddings` provider for Cloudflare Workers AI models whose upstream catalog metadata indicates text-embedding support. The provider callback posts embedding inputs to Cloudflare's account-scoped OpenAI-compatible `/embeddings` endpoint, so raw embedding input may leave the local machine for Cloudflare. It does not call `metidos.embeddings.embed(...)` as a consumer and does not store LanceDB vectors.

If future changes add vector storage, update the manifest and this file together: embedding consumers use `metidos:can_embed`; vector storage uses `metidos:lancedb` plus `storage:write` and must stay scoped to plugin-owned `~/` data.

## Context notes

Provider registration is initialization-only. Model discovery uses `network:fetch` against Cloudflare's official account-scoped model search API during provider refresh. Runtime chat inference is performed by Pi using the registered OpenAI-compatible provider configuration. Embedding requests are plugin callbacks that POST to `https://api.cloudflare.com/client/v4/accounts/{account_id}/ai/v1/embeddings` with bearer API token auth. Cloudflare account ID path validation is intentionally conservative so the plugin does not broaden network access beyond the declared account-scoped API paths.
