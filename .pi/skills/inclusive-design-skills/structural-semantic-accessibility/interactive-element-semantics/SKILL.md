# Interactive Element Semantics

Use this skill when evaluating interactive UI components. Focus on whether controls behave and announce themselves in ways users and assistive technologies expect.

## Review Component Meaning

- Check whether buttons, links, inputs, and widgets use appropriate native semantics
- Review accessible names, roles, values, states, and keyboard behavior
- Identify custom components that mimic controls without exposing correct semantics
- Consider discoverability, predictability, and consistency across the interface

## Improve Reliability

- Prefer native elements when possible
- Ensure custom components faithfully implement expected interaction and state exposure
- Match control type to user expectation: links navigate, buttons act, toggles change state
- Keep labels and announcements concise, specific, and stable

## Output

- Highlight semantic mismatches and interaction inconsistencies
- Recommend changes that improve compatibility and predictability
- Prioritize fixes for frequent and task-critical controls
