Getting started

Run Metidos locally from a clean clone.

This page summarizes the documented setup path only. Use it to orient yourself, then follow the canonical installation guide for exact commands, troubleshooting, backup, restore, and remote-access details.

Prerequisites

Install the basics first.

Metidos is an operator-owned local app. Start with a clean checkout and do not copy private runtime state from another machine.

  • Git, a modern browser, and the Bun version declared in package.json.
  • A private .env copied from .env.example; keep secrets out of commits, screenshots, logs, and issues.
  • A model provider credential or approved local/private provider is required for real agent turns. The UI can open before a provider is configured.

Clean clone path

Install, start, authenticate.

Use the full install guide when running these commands so you can check the current Bun requirement, app-data guidance, and troubleshooting notes.

  1. 1. Clone and install: clone the repository, enter it, then run bun install --frozen-lockfile.
  2. 2. Configure locally: copy .env.example to .env and keep provider keys or app-data paths private.
  3. 3. Start Metidos: run bun run start for a production-style local checkout, or bun run dev while developing the app.
  4. 4. Complete Local Auth: open the printed localhost URL, configure the first local operator login, and save recovery codes privately.

First safe workspace

Add a project before scheduling agents.

Keep the first run conservative: no private screenshots, no broad permissions, and no cron job until an interactive thread works.

  • Configure a providerUse Settings, environment variables, or an approved plugin-backed provider. Provider-free sessions should show setup guidance but cannot run model-backed turns.
  • Open a Project and WorktreeConfirm the displayed path is the disposable or intended checkout before starting agent work.
  • Start a safe ThreadLeave Unsafe Mode off and begin with a read-only request, such as asking Metidos to summarize test commands without editing files.
  • Review DiffsIf files change, review added, modified, deleted, renamed, and binary entries before keeping or committing anything.
