# Public asset provenance audit — 2026-06-02

This audit inventories checked-in binary visual/font assets that need ownership, provenance, and redistribution review before making the repository public.

## Scope and method

Command used:

```bash
git ls-files '*png' '*jpg' '*jpeg' '*svg' '*ico' '*webp' '*gif' '*woff2' '*ttf' '*otf' | sort
```

The inventory below covers only assets tracked by Git. Dependency assets under ignored or vendored package directories are out of scope unless they are copied into tracked repository paths. The inventory was updated after adding the generated website favicon/app-icon SVGs, website Open Graph preview source/export, and generated GitHub social preview source. The same inventory command was re-run on 2026-06-03 before the remaining human artwork decisions were available; tracked generated preview assets were added/confirmed, while the remaining human artwork decisions were still unchanged.

## Findings

- The tracked font assets have usable checked-in provenance and license files.
- The tracked artwork PNG assets do not yet have enough checked-in provenance to prove public redistribution rights.
- `src/mainview/crown.png` was removed on 2026-06-02 because no code referenced it and no checked-in provenance established public redistribution rights.
- `src/mainview/logo.png` was removed on 2026-06-02 because no code referenced it and no checked-in provenance established public redistribution rights.
- `src/mainview/pixel-crown.png` was inspected beyond the base inventory on 2026-06-02; no textual metadata or useful Git history was found for this app artwork asset, so its public-readiness decision remains blocked on maintainer confirmation.
- `docs/uploadthing-test.png` is a repo-owned generated 1×1 fixture and is safe to keep tracked for public redistribution.
- `docs/brand/github-social-preview.svg` is repo-owned generated GitHub social preview artwork with no external assets or private data and is safe to upload through GitHub repository settings.
- `docs/images/readme-feature-tour.svg` and `docs/images/readme-hero-demo.svg` are repo-owned generated README images with fake/demo data only and are safe to keep tracked for public redistribution.
- `bird.png` and `website/bird.png` are byte-identical duplicates. The root copy is used by the README, mainview favicon/social metadata, and backend mainview asset manifest. The website copy lets `website/` remain a self-contained static deployable folder for its header mascot, while the website favicon now uses generated SVG artwork. Both bird copies should remain tracked only if the maintainer confirms the shared mascot source is repo-owned or otherwise redistributable; otherwise replace/remove both before publishing.
- `website/favicon.svg`, `website/app-icon.svg`, `website/og.svg`, and `website/og.png` are repo-owned generated website icon/preview assets and are safe to keep tracked for public redistribution.
