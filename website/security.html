Security

Local-first software with operator-owned safety boundaries.

Metidos runs powerful coding agents on your machine. This page summarizes the public-safe security posture and links to the canonical repository security docs for exact implementation details.

Report a vulnerability Read security model

Posture

Treat Metidos like powerful local development software.

Metidos is designed for one Local Operator controlling one installation, not for anonymous multi-tenant hosting.

  • Local Auth protects browser access with first-run setup, sessions, WebSocket tickets, and step-up authentication for sensitive actions.
  • Safe Mode is the default posture for threads and cron jobs; Unsafe Mode should be enabled only when the operator intentionally needs broader tool access.
  • Backend services remain authoritative for auth, path scope, plugin lifecycle, provider auth, and permission boundaries.

Sensitive data

Keep secrets out of public channels.

Never share

API keys, OAuth tokens, cookies, session IDs, WebSocket tickets, TOTP secrets, recovery codes, full .env files, or App Data databases.

Sanitize before reporting

Use fake/demo data for screenshots and redact usernames, hostnames, private repositories, local absolute paths, branch names, customer data, and tokens.

Review generated output

Diagnostics and plugin logs can contain local context. Review them before sharing, especially when plugins or providers authored the content.

Boundaries

Plugins, providers, files, network, and cron all need explicit intent.

The repository docs contain the precise controls. This summary is a quick public website orientation.

  1. 1. Plugins are discovered before execution, require manifest review, and run only after Local Operator approval.
  2. 2. Plugin file and network access is declared through permissions, allowlists, and virtual roots.
  3. 3. Provider credentials belong in private runtime configuration or plugin secret settings, never in tracked source.
  4. 4. Cron jobs create future agent work, so review schedule, prompt, permissions, and unsafe-mode state before enabling them.
  5. 5. Remote access through a reverse proxy or Tailscale-style setup must preserve trusted origins, TLS, and private App Data boundaries.

Reporting

Report suspected vulnerabilities privately.

Do not open a public issue for a suspected vulnerability until there is a coordinated disclosure plan.

Use the private reporting channel documented in SECURITY.md. Include the affected commit or version, OS, Bun version, component, reproduction steps, impact, and sanitized evidence when safe.

For non-sensitive help, use the public support guidance in SUPPORT.md.
