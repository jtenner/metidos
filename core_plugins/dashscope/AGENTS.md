# AGENTS for Alibaba Cloud Model Studio (DashScope)

## Purpose

First-party provider plugin for Alibaba Cloud Model Studio / DashScope. It registers Plugin System v1 model providers for Qwen chat models and DashScope text embedding models. Runtime chat inference is handed to Pi's OpenAI-compatible transport for one configured fixed regional endpoint; embedding requests are sent by this plugin to DashScope's OpenAI-compatible `/embeddings` endpoint for supported regions.

## Source layout

- `metidos-plugin.json`: v1 manifest reviewed by the local operator; declares `DASHSCOPE_API_KEY`, `QWEN_API_KEY`, the optional `api_key` plugin setting, the `region` enum setting, narrow embedding network access, and the `dashscope` / `dashscope_embeddings` providers.
- `index.ts`: plugin entry point that returns the static region-scoped model catalog and implements the embedding provider callback.
- `AGENTS.md`: this guide.
- `.data/`: generated plugin data owned by Metidos; this plugin does not intentionally create files there.
- `.logs/`: generated plugin logs if future changes add `log:write`; this plugin does not currently request logging and should not log prompts, completions, embedding input, vectors, API keys, or Authorization headers.
- `.data-bak-*`: generated reset backups; do not commit.

Root `node_modules/` is forbidden.

## Validation

1. Validate `metidos-plugin.json` against the Metidos Plugin Manifest v1 schema.
2. Run `bun test src/bun/plugin/dashscope-core-plugin.test.ts` from the repository root.
3. Confirm no root `node_modules/` exists.
4. Confirm imports are local or `@metidos/plugin-api`.

## `.data` contents

This plugin has no durable plugin-owned data. If `.data/` exists, it is runtime-created by Metidos and can be regenerated by Reset Plugin Data.

## Safe `.data` inspection

- Prefer read-only inspection.
- Do not copy or print secret-bearing files if future changes add local credentials.
- Do not edit `.data` while the plugin sidecar is running unless this guide is updated with an explicit repair workflow.

## Safe `.data` repair

There are no hand-repairable data files today. Prefer Metidos Reset Plugin Data for unknown `.data` contents; it should be safe because this plugin has no seed data or durable local state.

## Reset behavior

Metidos Reset Plugin Data moves `.data` to `.data-bak-{timestamp}`, recreates `.data`, and restarts/reloads the plugin. No local configuration is recreated because credentials live in plugin settings, `DASHSCOPE_API_KEY`, or `QWEN_API_KEY`.

## Secrets and logs

Secrets are provided through the optional secret plugin setting `api_key` or the secret env vars `DASHSCOPE_API_KEY` and `QWEN_API_KEY`. Credential precedence for discovery, embeddings, and inference is plugin `api_key`, then `DASHSCOPE_API_KEY`, then `QWEN_API_KEY`, as declared in the provider configuration `piAuth` records.

The current manifest does not request `log:write`. If future changes add logging, do not log API keys, Authorization headers, prompts, completions, request bodies, embedding input, embedding vectors, model responses, or raw provider payloads. Bounded status/error messages are acceptable only if they exclude secrets and user content.

## Embeddings and vector search

This plugin provides embedding models through `metidos:provides_embeddings` and the `dashscope_embeddings` provider callback in `index.ts`. Embedding-capable models are marked with `api: "embeddings"` and `compat.providesEmbeddings: true`. The callback sends non-empty text input to the configured region's DashScope OpenAI-compatible `/embeddings` endpoint and returns the first finite numeric vector from the response.

Raw embedding input leaves the local machine and is sent to Alibaba Cloud Model Studio. The plugin does not call `metidos.embeddings.embed(...)`, does not consume embeddings from the host, does not request `metidos:can_embed`, and does not store LanceDB vectors. If future changes add vector storage, update the manifest and this file together: vector storage uses `metidos:lancedb` plus `storage:write` and must stay scoped to plugin-owned `~/` data.

## Context notes

Provider registration is initialization-only. Runtime chat inference is performed by Pi using the registered OpenAI-compatible provider configuration. This plugin intentionally limits region selection to fixed official endpoint hosts so the manifest can keep a narrow network allowlist and avoid arbitrary user-provided base URLs. US-region embeddings are not exposed because the verified embedding docs list supported regions separately from the US chat endpoint.
