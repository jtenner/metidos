# Visual Accessibility Design

Use this skill when evaluating visual interfaces for people with a range of vision needs. Focus on legibility, distinguishability, structure, and whether users can succeed without perfect sight.

## Assess Visual Demands

- Check text size, contrast, spacing, zoom/reflow behavior, and color reliance
- Review icons, focus indicators, controls, and charts for visibility and meaning
- Consider low vision, color blindness, glare, fatigue, and temporary visual constraints
- Identify content that becomes unusable under magnification or narrow viewports

## Improve Perception

- Strengthen visual hierarchy and interactive affordances
- Add labels, patterns, or text equivalents where visuals alone are ambiguous
- Support zoom and text resizing without clipping or overlap
- Ensure core tasks remain possible with assistive technologies and non-visual cues

## Output

- Highlight weak visibility and readability patterns
- Recommend design changes that improve perception across contexts
- Prioritize fixes for essential navigation, input, and feedback
