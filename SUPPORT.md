# Support

Metidos is early-stage local developer tooling. Please keep support requests focused on repository issues, installation problems, documentation gaps, and reproducible bugs.

## Usage questions

Open a GitHub Discussion if Discussions are enabled. If Discussions are not enabled, open a documentation issue and describe what you were trying to learn.

## Bugs

Use the bug report issue template. Include:

- Metidos version or commit,
- operating system and version,
- Bun version,
- command you ran,
- expected behavior,
- actual behavior,
- minimal reproduction steps,
- sanitized logs or screenshots when helpful.

## Installation problems

Use the install problem issue template. Include your OS, Bun version, install command, clean-clone status, and sanitized logs.

## Plugin issues

Use the plugin issue template. Include the plugin name, manifest details, requested permissions, lifecycle state, relevant sanitized logs, and whether Unsafe Mode was involved.

## Security vulnerabilities

Do not open a public issue for vulnerabilities or suspected credential exposure. Follow [`SECURITY.md`](SECURITY.md) and email responsible disclosure reports to <tenner.joshua@gmail.com>.

## Redaction reminder

Do not paste secrets, provider keys, OAuth tokens, recovery codes, session cookies, full `.env` files, private repository URLs, local database files, plugin `.data`, plugin `.logs`, or screenshots containing private data.
