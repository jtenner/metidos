# AGENTS for Volcengine Ark

## Purpose

This first-party core plugin registers ByteDance Volcengine Ark / Doubao as a Metidos model provider. It exposes a conservative static catalog of documented OpenAI-compatible Doubao chat models and hands API-key authentication to Pi.

Registered capabilities:

- `provider:register` for the `volcengine` model provider family.

The plugin does not perform plugin-owned network fetches. Pi performs model inference against `https://ark.cn-beijing.volces.com/api/v3` after the local operator configures an API key.

## Source layout

- `metidos-plugin.json`: v1 manifest reviewed by the local operator.
- `index.ts`: plugin entry point and static Doubao model catalog.
- `AGENTS.md`: this guide.
- `.data/`: generated plugin data owned by Metidos; this plugin does not intentionally create data files.
- `.logs/`: generated plugin logs when enabled; this plugin does not intentionally write logs.
- `.data-bak-*`: generated reset backups; do not commit.

Root `node_modules/` is forbidden.

## Validation

From the repository root:

1. Validate the manifest with the Metidos schema if editing manifest fields.
2. Run `bun test src/bun/plugin/volcengine-core-plugin.test.ts`.
3. Run the repository workflow: `bun format` then `bun validate` before committing code changes.
4. Confirm no root `node_modules/` exists.
5. Confirm imports are local or `@metidos/plugin-api` only.

## `.data` contents

This plugin has no intentional durable `.data` files. Any `.data` contents are generated by the Metidos plugin runtime and should be treated as runtime-owned.

## Safe `.data` inspection

- Prefer read-only inspection through Metidos Settings.
- Do not copy or print secret-bearing runtime files unless needed for repair.
- Do not edit `.data` while the plugin sidecar is running unless future plugin docs explicitly allow it.

## Safe `.data` repair

There are no documented hand-editable data files for this plugin. If runtime data appears corrupt, disable the plugin and use Metidos Reset Plugin Data instead of manual edits.

## Reset behavior

Metidos Reset Plugin Data moves `.data` to `.data-bak-{timestamp}`, recreates `.data`, and restarts/reloads the plugin. Because this plugin has no seed data, reset does not restore credentials; API keys remain configured through Plugin Settings or host environment variables.

## Secrets and logs

Secrets are supplied through the `api_key` Plugin Setting, `ARK_API_KEY`, or `VOLCENGINE_API_KEY`. Do not put API keys in source files, docs, logs, thrown errors, model names, or commit messages.

The plugin does not intentionally call `metidos.log` and does not request `log:write`. Metidos v1 does not promise automatic redaction of plugin-authored text, so future logging changes must avoid secrets, prompts, completions, request bodies, Authorization headers, and API keys.

## Embeddings and vector search

This plugin does not provide embedding models, consume embeddings, or store LanceDB vectors.

Do not add `metidos:provides_embeddings` unless an `embed(context, request)` callback is implemented and tested with finite numeric vectors. Do not add `metidos:can_embed` or `metidos:lancedb` unless the plugin becomes an embedding consumer or vector store.

## Context notes

- Provider registration is initialization-only.
- The plugin has no `./` project file access, no terminal access, no crons, and no plugin-owned network access.
- Inference is delegated to Pi's OpenAI-compatible completions transport using the configured base URL and auth handoff.
- The static catalog intentionally avoids arbitrary user-provided base URLs, regional endpoint switching, endpoint-management APIs, and non-chat Ark APIs in this safe core slice.
