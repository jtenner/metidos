The problem

Agent power needs visible boundaries.

AI agents can run commands, edit files, schedule work, and call plugins. That power is useful only when you can see what is running, what it can access, and what changed before you trust the result.

Metidos keeps agent threads, plugin permissions, scheduled jobs, and diffs in one local workspace you can inspect and control.

Local-first by design

Your machine, your rules.

Metidos runs on loopback and treats the operator as the source of truth. Risky actions are gated, not assumed.

Local authentication
First-run auth on a loopback server — no hosted account, no multi-tenant backend.
Step-up checks
Sensitive operations require an explicit step-up confirmation before they run.
Plugin review hashes
Plugins are approved by content hash (sha256:9f2c…) so changes re-trigger review.
Scoped permissions
Filesystem and network access are scoped per plugin and per thread, not granted blanket.
Operator review
High-risk actions wait for operator review before execution. Unsafe mode is opt-in, never default.

Architecture

A small, legible stack.

Local persistence

UI and backend talk over typed WebSocket RPC, so the workspace stays responsive while work runs in the background.

Status — alpha

Metidos is pre-1.0 local developer tooling. Expect rough edges, changing APIs, and incomplete release polish. Keep backups of important local data, review plugin and provider access carefully, and don't treat Unsafe Mode or unreviewed plugins as safe defaults.
