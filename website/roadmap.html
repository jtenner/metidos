Roadmap

Project status and hardening priorities.

Metidos is being prepared for public, local-first use. This page summarizes tracked repository direction without launch promotion, private deployment details, or speculative dates.

Repository roadmap Browse docs

Current status

Local app, public-readiness work in progress.

The repository contains the Bun backend, React Mainview, Pi-powered agent runtime integration, plugin documentation, install guidance, and static website pages.

  • Local-first command centerProjects, worktrees, threads, diffs, cron jobs, settings, and plugin administration are organized around an operator-owned local workspace.
  • Safety model documentedLocal Auth, unsafe-mode boundaries, plugin approvals, provider credentials, diagnostics, backup, and disclosure paths have public documentation.
  • Public launch checklist activeRemaining repository work focuses on provenance, clean-install verification, screenshots with fake data, tests, and GitHub repository settings.

Near-term hardening

Make everyday operation easier to verify.

The next work slices should be small, reviewable, and tied to concrete validation evidence.

  • Expand test coverage around terminal workspaces, thread composition, settings, plugin administration, and auth/session side effects.
  • Verify major product flows with disposable projects, fake providers, and explicit pass/fail notes.
  • Improve diagnostics export and error guidance while keeping secrets, tokens, private files, and provider keys out of reports.

Plugin ecosystem

Grow extensions without weakening operator control.

Plugin work should preserve the review-first model: declared capabilities, visible source changes, narrow approvals, and clear reset paths.

  • Website plugin overviewLocal review-first plugin safety model and approval flow
  • Plugin System v1Architecture, manifest model, access groups, and host integration
  • Plugin authoring guidePractical guidance for safe local plugin development

Install and first run

Clean setup evidence comes before polish.

The install path should be validated from disposable environments before public claims become more specific.

  1. 1. Clean clone: verify documented setup, dependency installation, startup, and validation commands on a disposable checkout.
  2. 2. Local Auth: verify first-user setup, login, logout, recovery, reset, and pending-user states without recording secrets or recovery codes.
  3. 3. Provider expectations: document provider-free and fake-provider behavior so users know what works before adding credentials.
  4. 4. Backup and restore: smoke-test disposable App Data procedures and make failure states readable.

Known limitations

Assumptions to confirm before relying on Metidos.

  • Metidos is designed for local operators; exposing it beyond localhost requires careful reverse-proxy, auth, and backup review.
  • Real agent turns require configured model-provider access or an approved local/private provider.
  • Some public-readiness evidence, artwork provenance decisions, screenshots, and clean-environment smoke tests are still being completed.
  • Roadmap items may change as tests, audits, plugin needs, and release validation uncover better priorities.
