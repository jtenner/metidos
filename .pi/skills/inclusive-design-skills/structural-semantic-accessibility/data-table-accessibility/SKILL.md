# Data Table Accessibility

Use this skill when evaluating tabular content. Focus on whether users can understand relationships between rows, columns, headers, controls, and states across visual and assistive-technology use.

## Review Table Usability

- Check header associations, captions, summaries, and cell meaning
- Review sorting, filtering, selection, pagination, and responsive table patterns
- Identify tables that become confusing when linearized, zoomed, or collapsed on mobile
- Consider dense data, sticky elements, and keyboard navigation complexity

## Improve Access & Comprehension

- Use semantic table structure for real tabular data
- Keep headers clear and repeated context available where needed
- Provide alternative views or summaries when tables become unwieldy on small screens
- Ensure controls within or around tables are labeled and keyboard operable

## Output

- Highlight structural, semantic, or usability issues in table design
- Recommend changes that improve comprehension and navigation
- Prioritize fixes for high-value data workflows and decision-making tasks
