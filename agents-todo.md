## Risks

- Hosted-provider compatibility risk. The Pi runtime adapter now runs real Jolt thread and cron lifecycles through Bun successfully, but the committed coverage is still probe-backed plus focused smoke tests. Real hosted-provider auth, transport quirks, and long-running tool sessions still need follow-up coverage. See [1. Runtime Integration](./docs/2026-04-09-pi-coding-agent-migration-research.md#1-runtime-integration), [RT02 implementation status in `jt-ide`](./docs/2026-04-09-pi-coding-agent-migration-research.md#rt02-implementation-status-in-jt-ide), [CR12 implementation status in `jt-ide`](./docs/2026-04-09-pi-coding-agent-migration-research.md#cr12-implementation-status-in-jt-ide), and [Recommended Phased Plan](./docs/2026-04-09-pi-coding-agent-migration-research.md#recommended-phased-plan).
- Safety semantics risk. `unsafeMode` now has an explicit Pi-era meaning in code: safe threads keep worktree-scoped file/search/edit/write tools and cannot escalate Jolt-created child threads or cron jobs to unsafe mode, while unsafe threads also gain `bash` and unsafe child creation. That closes the biggest ambiguity, but it is still not equivalent to Codex sandbox/network semantics or a real container boundary. See [RT02 implementation status in `jt-ide`](./docs/2026-04-09-pi-coding-agent-migration-research.md#rt02-implementation-status-in-jt-ide), [SA09 implementation status in `jt-ide`](./docs/2026-04-09-pi-coding-agent-migration-research.md#sa09-implementation-status-in-jt-ide), [No built-in Codex-style sandbox policy](./docs/2026-04-09-pi-coding-agent-migration-research.md#no-built-in-codex-style-sandbox-policy), and [6. Access Controls and Unsafe Mode](./docs/2026-04-09-pi-coding-agent-migration-research.md#6-access-controls-and-unsafe-mode).
- Feature parity risk. GitHub now has a real Pi-era tool pack and `agentsAccess` now has a real Pi-era meaning, but both are intentionally narrower than old Codex parity. GitHub remains read-heavy and current-repository-scoped, while the agents pack stops at `update_plan` plus one-shot `delegate_task` instead of a persistent child-agent lifecycle or interactive plan-mode UI. See [GH10 implementation status in `jt-ide`](./docs/2026-04-09-pi-coding-agent-migration-research.md#gh10-implementation-status-in-jt-ide), [AG11 implementation status in `jt-ide`](./docs/2026-04-09-pi-coding-agent-migration-research.md#ag11-implementation-status-in-jt-ide), [Pi Non-Goals That Matter for Jolt](./docs/2026-04-09-pi-coding-agent-migration-research.md#pi-non-goals-that-matter-for-jolt), [7. GitHub Integration](./docs/2026-04-09-pi-coding-agent-migration-research.md#7-github-integration), and [8. Agents and Plan Mode](./docs/2026-04-09-pi-coding-agent-migration-research.md#8-agents-and-plan-mode).
- Telemetry presentation risk. TM13 now exposes Pi-backed context usage, compaction history, live phase, and queue counts through Jolt’s thread detail and status payloads, but the browser currently only consumes the refreshed context meter. There is still no explicit UI for `runStatus.phase`, `runStatus.queue`, Pi cost totals, or cache-write metrics. See [TM13 implementation status in `jt-ide`](./docs/2026-04-09-pi-coding-agent-migration-research.md#tm13-implementation-status-in-jt-ide), [10. Telemetry, Usage, and Compaction UI](./docs/2026-04-09-pi-coding-agent-migration-research.md#10-telemetry-usage-and-compaction-ui), and [Phase 6](./docs/2026-04-09-pi-coding-agent-migration-research.md#recommended-phased-plan).
- Transcript parity risk. TL07 restores minimal `file_change` parity for successful Pi `edit` and `write` calls, but `web_search` and any richer Pi-extension-specific transcript kinds are still unresolved. See [TL07 implementation status in `jt-ide`](./docs/2026-04-09-pi-coding-agent-migration-research.md#tl07-implementation-status-in-jt-ide), [Current Codex-Specific Surface In Jolt](./docs/2026-04-09-pi-coding-agent-migration-research.md#current-codex-specific-surface-in-jolt), and [5. Transcript/Event Mapping](./docs/2026-04-09-pi-coding-agent-migration-research.md#5-transcriptevent-mapping).
- Pi session lifecycle risk. DB03 now persists `piSessionId`, `piSessionFile`, and `piLeafEntryId`, but Jolt still only stores references to Pi’s JSONL session system and does not yet expose or stress-test branching, recovery, or UI-level session navigation. See [DB03 implementation status in `jt-ide`](./docs/2026-04-09-pi-coding-agent-migration-research.md#db03-implementation-status-in-jt-ide), [3. Pi already has a full session system](./docs/2026-04-09-pi-coding-agent-migration-research.md#3-pi-already-has-a-full-session-system), and [2. Persistence and Thread Identity](./docs/2026-04-09-pi-coding-agent-migration-research.md#2-persistence-and-thread-identity).
- Provider auth and availability risk. UI05 now exposes provider-qualified model labels and model-specific thinking support in the browser, but the app still has no provider login, auth-health, or availability hints for users choosing across Pi-backed endpoints. See [UI05 implementation status in `jt-ide`](./docs/2026-04-09-pi-coding-agent-migration-research.md#ui05-implementation-status-in-jt-ide), [3. Provider/Model UI Migration](./docs/2026-04-09-pi-coding-agent-migration-research.md#3-providermodel-ui-migration), and [Migration Requirement Matrix](./docs/2026-04-09-pi-coding-agent-migration-research.md#migration-requirement-matrix).
- Browser extension UI parity risk. UX14 now binds Pi extension dialogs, notifications, status lines, widgets, working-message overrides, hidden-thinking labels, and editor text updates into the browser app, but Pi’s TUI-only surfaces such as `custom(...)`, custom header/footer components, terminal-input listeners, and custom editor components remain intentionally unsupported in Jolt. See [UX14 implementation status in `jt-ide`](./docs/2026-04-09-pi-coding-agent-migration-research.md#ux14-implementation-status-in-jt-ide), [12. Browser UI Bindings for Pi Extensions](./docs/2026-04-09-pi-coding-agent-migration-research.md#12-browser-ui-bindings-for-pi-extensions), and [Pi Non-Goals That Matter for Jolt](./docs/2026-04-09-pi-coding-agent-migration-research.md#pi-non-goals-that-matter-for-jolt).
