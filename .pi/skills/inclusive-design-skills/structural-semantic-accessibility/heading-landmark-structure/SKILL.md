# Heading & Landmark Structure

Use this skill when assessing navigational structure. Focus on whether users can jump through the page efficiently, understand major regions, and build a mental model of the content.

## Review Navigability

- Check heading levels, landmark roles, page titles, and region labels
- Identify skipped heading levels, duplicated labels, or missing structural cues
- Consider how screen reader users move by heading or landmark through the page
- Review whether visual and semantic structure match each other

## Improve Structure

- Use headings to reflect true content hierarchy, not just styling
- Provide meaningful landmarks and labels for major page regions
- Avoid unnecessary regions that add noise to navigation
- Keep structure consistent across similar pages and templates

## Output

- Highlight structural navigation issues
- Recommend heading and landmark improvements that improve orientation
- Prioritize changes that reduce navigation effort and ambiguity
